## Problem

Need the client-side JavaScript controller that handles form submission, progress polling, result rendering, and copy-to-clipboard functionality for the character brainstormer page.

## Assumption Reassessment (2026-03-25)

1. Client controllers follow the pattern in `16-character-webs-controller.js`: `initXxxPage()` function, DOM element caching, state object, event listeners, AJAX calls.
2. Progress polling uses the existing polling pattern with `generationProgressService` and stage display.
3. Copy-to-clipboard uses `navigator.clipboard.writeText()` with "Copied!" feedback (1500ms), pattern from `15a-stage-renderers.js`.
4. Files are numerically prefixed for concat order. `17` and `18` are taken (`17-characters-controller.js`, `18-worldbuilding-controller.js`), so the next available is `19-character-brainstormer-controller.js`.
5. After creating the controller, `node scripts/concat-client-js.js` must be run to regenerate `app.js`.

## Architecture Check

1. Follows existing controller patterns exactly — no novel client-side architecture.
2. Plain vanilla JS (no framework) consistent with the rest of the codebase.
3. No backwards-compatibility shims needed.

## What to Change

### 1. Create client controller

File: `public/js/src/19-character-brainstormer-controller.js` (or next available number)

Functions:
- `initCharacterBrainstormerPage()` — Entry point, called on page load. Checks for page element, caches DOM refs, initializes state and listeners.
- `handleBrainstormGenerate()` — POST to `/character-brainstormer/api/generate` with form data + API key. Start progress polling. On success, call `renderBrainstormResults()`.
- `renderBrainstormResults(result)` — Show results section, render diversity note, render character cards with all fields.
- `renderCharacterCard(character)` — Returns HTML string for a single character card with "Copy as Markdown" button.
- `copyCharacterAsMarkdown(character, buttonEl)` — Format single character as markdown, copy to clipboard, show "Copied!" feedback.
- `copyAllCharactersAsMarkdown(characters, diversityNote, buttonEl)` — Format all characters as markdown, copy to clipboard.
- `characterToMarkdown(character)` — Returns markdown string for a single character.

### 2. Progress polling

Use existing progress polling utility (if available) or implement polling to `GET /play/:storyId/progress/:progressId` — but since this is a standalone page (not play page), check if the progress polling pattern is generic or play-specific. If play-specific, implement a simple poll to the brainstormer's progress endpoint.

Note: The route handler (CHABRA-005) must expose a progress endpoint or use the generic `generationProgressService`. The controller polls accordingly.
