# CHABIBSTASPL-006: Register new chat generation stages and update metadata

**Status**: PENDING
**Priority**: HIGH
**Effort**: Small
**Engine Changes**: Yes — stage registry, generation stages enum, client-side stage metadata
**Deps**: None (can be done early; required by CHABIBSTASPL-004 at runtime and CHABIBSTASPL-005 for pipeline)

## Problem

The new LLM stages (`chatSceneContext`, `chatCharacterContext`) and generation stages (`CURATING_CHAT_SCENE`, `CURATING_CHAT_CHARACTER`) need to be registered in the stage registry, metadata JSON, and auto-generated client files. The old `chatBible` / `CURATING_CHAT_BIBLE` entries must be removed.

## Assumption Reassessment (2026-03-27)

1. `LLM_STAGE_KEYS` array in `src/config/llm-stage-registry.ts` includes `'chatBible'` — **confirmed**.
2. `GENERATION_STAGES` in `src/engine/generated-generation-stages.ts` is auto-generated from `src/config/generation-stage-metadata.json` by `scripts/sync-stage-metadata.js` — **confirmed**.
3. `generation-stage-metadata.json` has an entry for `CURATING_CHAT_BIBLE` with display name and flavor phrases — **confirmed**.
4. Client-side stage metadata (`public/js/src/00-stage-metadata.js`) is auto-generated from the same JSON — **confirmed**.
5. `public/js/app.js` is regenerated by `node scripts/concat-client-js.js` — **confirmed**.
6. The sync script must be run after modifying the metadata JSON: `node scripts/sync-stage-metadata.js` — **confirmed**.

## Architecture Check

1. Single-source-of-truth approach: modify `generation-stage-metadata.json`, run sync script, run concat script. No manual duplication across files.
2. No backwards-compatibility aliasing. Old entries are replaced, not deprecated.

## What to Change

### 1. Update `src/config/llm-stage-registry.ts`

- Remove `'chatBible'` from `LLM_STAGE_KEYS`
- Add `'chatSceneContext'` and `'chatCharacterContext'` to `LLM_STAGE_KEYS`

### 2. Update `src/config/generation-stage-metadata.json`

Remove the `CURATING_CHAT_BIBLE` entry. Add two new entries:

```json
{
  "CURATING_CHAT_SCENE": {
    "displayName": "CURATING CHAT SCENE",
    "phrasePools": [
      "Mapping the physical space and recent pressures...",
      "Establishing what is true about the scene right now...",
      "Distilling the situation into hard scene facts..."
    ]
  },
  "CURATING_CHAT_CHARACTER": {
    "displayName": "CURATING CHAT CHARACTER",
    "phrasePools": [
      "Reading the character's emotional state against the scene...",
      "Synthesizing relationship dynamics and knowledge boundaries...",
      "Deciding what the character wants, fears, and will protect..."
    ]
  }
}
```

### 3. Run sync and concat scripts

```bash
node scripts/sync-stage-metadata.js
node scripts/concat-client-js.js
```

This auto-updates:
- `src/engine/generated-generation-stages.ts`
- `public/js/src/00-stage-metadata.js`
- `public/js/app.js`

### 4. Update stage model configuration

If `src/config/stage-model.ts` (or equivalent) maps stage keys to model selections, add entries for `chatSceneContext` and `chatCharacterContext` (same model as `chatBible` used). Remove `chatBible` entry.
